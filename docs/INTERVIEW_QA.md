# 📝 사업자등록 후 필요 기능 - 인터뷰 Q&A 기록

> **인터뷰 일시**: 2026년 1월 25일  
> **목적**: 사업자등록 후 구현 예정인 기능들에 대한 상세 요구사항 수집

---

## 1. 우선순위 및 기능 범위

### Q1: 사업자등록 후 구현 예정 기능 중 가장 우선순위가 높은 것은?
**답변**: 모두 중요 (우선순위 정하기 어려움)

### Q2: 한정된 리소스를 가정할 때 구현 순서는?
**답변**:
1. **1순위**: 카카오 알림톡 - 서비스 핵심 플로우(UX) 개선
2. **2순위**: SMS 본인인증 - 보안성 강화
3. **3순위**: PDF Storage - 문서 영구 보관

---

## 2. 카카오 알림톡 상세

### Q3: 카카오 알림톡을 발송할 시점은?
**답변**: 사장님이 계약서를 전송했을 때 (근로자에게)

### Q4: 알림톡 발송을 위한 서비스 제공사는?
**답변**: 알리고 (Aligo) - 저렴하고 국내 많이 사용

### Q5: 알림톡 발송 실패 시 대체 방안은?
**답변**: 일반 SMS로 자동 대체 발송 (Fallback)

### Q6: 카카오 알림톡 템플릿 등록/승인 과정에 대한 경험이나 계획은?
**답변**: 가이드가 필요함

### Q7: 알림톡 발송 비용(건당 약 8~15원)을 어떻게 처리할 계획인가요?
**답변**: 회사가 비용 부담 (운영비)

### Q8: 알림톡 메시지 톤은?
**답변**: 앱 내 UX Writing과 동일하게 (친근함/해요체)

### Q9: 알림톡 메시지에 포함할 버튼 개수는?
**답변**: 1개 - "계약서 확인하기"만

### Q10: 현재 카카오톡 SDK 공유 기능과 알림톡을 어떻게 공존시킬까요?
**답변**: 알림톡으로 SDK 공유 대체

---

## 3. SMS 본인인증 상세

### Q11: SMS 본인인증을 언제 사용할 계획인가요?
**답변**: 회원가입 시 (전체 사용자)

### Q12: 현재 MVP의 '휴대폰 번호 매칭' 방식과 SMS 인증의 관계는?
**답변**: SMS 인증으로 완전 교체

### Q13: SMS 발송 서비스는 알림톡과 동일한 알리고를 사용할 계획인가요?
**답변**: 예, 알리고에서 알림톡/SMS 모두 처리

### Q14: SMS 인증번호의 유효 시간은?
**답변**: 3분 (보통의 기준)

### Q15: SMS 인증번호 재발송 제한은?
**답변**: 시간당 3회 제한

### Q16: SMS 인증번호는 몇 자리로 할까요?
**답변**: 4자리 (간편)

### Q17: 인증번호 오류 입력 횟수 제한은?
**답변**: 3회 실패 시 재발송 필요

### Q18: SMS 인증 화면은 어디에 배치할까요?
**답변**: 모름 / 아직 결정 안됨

### Q19: SMS 인증 성공 후 플로우는?
**답변**: 모름 / 아직 결정 안됨

### Q20: SMS 발송 비용(건당 약 15~20원)에 대한 우려가 있나요?
**답변**: 없음 - 회사가 비용 부담

## 4. PDF Storage 상세

### Q21: PDF를 Supabase Storage에 저장하는 시점은?
**답변**: 양측 서명 완료 시 + 추후 수정 시 재생성

### Q22: PDF 보관 기간은?
**답변**: 5년 보관 (근로기준법 기준)

### Q23: PDF 다운로드 방식은?
**답변**: 미리보기 후 다운로드 버튼

### Q24: PDF 수정 시 버전 관리가 필요할까요?
**답변**: 아니오 - 최신 PDF로 덮어쓰기
